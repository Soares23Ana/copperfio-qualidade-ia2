## Épico 1 — Autenticação e Gestão de Conta
**Descrição: Gerenciamento do ciclo de vida da conta do usuário, garantindo a criação, o acesso seguro e a recuperação de credenciais de forma prática.**

### User Stories
**US01 — Cadastrar-se Como visitante, quero me cadastrar informando meus dados e criando uma senha para ter uma conta no aplicativo.**

**US02 — Realizar Login Como usuário, quero digitar meu e-mail e senha na tela de acesso para entrar na minha conta com segurança.**

**US03 — Recuperar e Redefinir Senha Como usuário que esqueceu a senha, quero digitar meu e-mail e receber um link de recuperação para criar e redefinir minha nova senha.**

## Épico 2 — Catálogo e Informações de Produtos/Serviços
**Descrição: Área dedicada à apresentação dos itens disponíveis no aplicativo, permitindo que o usuário explore e conheça detalhes técnicos do que está sendo oferecido.**

### User Stories
**US10 — Acessar Catálogo Como usuário, quero visualizar a lista do catálogo para conhecer os itens disponíveis.**

**US11 — Visualizar Fichas Técnicas Como usuário, quero clicar em um item e abrir sua ficha técnica para ler informações mais detalhadas sobre ele.**

**US12 — Buscar Itens Como usuário, quero utilizar uma barra de pesquisa no catálogo para encontrar um item específico mais rapidamente.**

## Épico 3 — Central de Chamados e Reclamações
**Descrição: Canal de suporte direto onde o cliente pode relatar problemas operacionais e enviar solicitações estruturadas para a equipe de atendimento.**

### User Stories
**US20 — Acessar a Central de Suporte Como cliente, quero abrir a central de chamados/reclamações para buscar ajuda sobre um problema.**

**US21 — Descrever a Reclamação Como cliente, quero preencher um campo com a descrição detalhada do ocorrido para que a equipe entenda exatamente o meu problema.**

**US22 — Enviar Chamado Como cliente, quero enviar o chamado preenchido no sistema para que o suporte receba e inicie a tratativa da minha reclamação.**

## Épico 4 — Avaliação de Experiência e Feedback
**Descrição: Ferramentas criadas para coletar a opinião do cliente sobre a experiência de uso do aplicativo, gerando insights para melhorias contínuas.**

### User Stories
**US30 — Avaliar Experiência Como cliente, quero dar uma nota ou classificar minha experiência no app para demonstrar meu nível de satisfação.**

**US31 — Enviar Feedback Como cliente, quero escrever um texto livre com meu feedback ou sugestões para ajudar a empresa a melhorar o serviço.**

**US32 — Feedback Anônimo Como cliente, quero ter a opção de enviar meu feedback de forma anônima para manter minha privacidade, caso eu deseje.**

## Épico 5 — Dashboard do Gestor
**Descrição: Painel administrativo e gerencial exclusivo para a equipe interna acompanhar as solicitações dos clientes e a saúde do negócio.**

### User Stories
**US40 — Acessar Dashboard Como gestor, quero fazer login e ser direcionado ao dashboard principal para ter uma visão gerencial do aplicativo.**

**US41 — Gerenciar Chamados e Reclamações Como gestor, quero visualizar a lista de todos os chamados enviados pelos clientes para que eu possa acompanhar e resolver os problemas relatados.**

**US42 — Analisar Feedbacks Como gestor, quero ler as avaliações e os feedbacks dos clientes no meu painel para entender o que precisa ser melhorado na operação.**

## Épico 6 — Comunicação e Mural de Avisos
**Descrição: Mecanismo de comunicação para manter a base de usuários informada sobre novidades, atualizações de sistema ou comunicados gerais.**

### User Stories
**US50 — Visualizar Avisos Como usuário, quero acessar o mural de avisos para me manter informado sobre comunicados importantes do aplicativo.**

**US51 — Publicar Avisos Como gestor, quero criar e publicar novos avisos no mural para repassar informações urgentes ou novidades aos clientes.**

**US52 — Gerenciar Avisos Como gestor, quero editar ou excluir avisos antigos para manter o mural sempre organizado e com informações atuais.**

# Observações
Cada épico pode ter quantas stories forem necessárias.

O backlog inicial contém apenas épicos + primeiras stories (alto nível).

Stories devem sempre seguir o modelo:

Como [ator], quero [intenção] para [benefício].
